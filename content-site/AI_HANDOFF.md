## Why Upload Packaging Exists

Docusaurus is configured with:

```ts
url: 'https://allanbpediniv.com',
baseUrl: '/info/',
```

Docusaurus still writes generated files directly into `content-site/build/`, for example `build/index.html` and `build/assets/...`. Cloudflare Pages serves uploaded files relative to the domain root. If `build/` is uploaded directly, `https://allanbpediniv.com/info/` returns 404 because `index.html` is at the upload root instead of under `info/`.

The workflow fixes that by packaging the build output like this:

```text
content-site/cloudflare-pages/
  _headers
  _redirects
  info/
    index.html
    assets/
    img/
    research/
    insights/
    newsroom/
```

The workflow step that does this is `Prepare Cloudflare Pages artifact` in `.github/workflows/deploy.yml`.

## Routing And Headers

`content-site/static/_redirects` contains:

```text
/ /info/ 301
```

During CI this is copied to `content-site/cloudflare-pages/_redirects`, so Cloudflare sees it at the upload root.

`content-site/static/_headers` contains:

```text
/info/assets/*
  Cache-Control: public, max-age=31536000, immutable
```

During CI this is copied to `content-site/cloudflare-pages/_headers`.

## GitHub Actions Setup

The workflow is `Deploy to Cloudflare Pages`.

It runs on:

- Pushes to `main` when `content-site/**` or `.github/workflows/deploy.yml` changes.
- Pull requests touching the same paths.
- Manual `workflow_dispatch`.

Build job:

1. Checks out the repo.
2. Installs Node 20.
3. Runs `npm ci` in `content-site/`.
4. Runs `npm run typecheck`.
5. Runs `npm run build`.
6. Packages `build/` under `cloudflare-pages/info/`.
7. Uploads `cloudflare-pages/` as the deploy artifact.

Deploy job:

1. Downloads the artifact to `content-site/cloudflare-pages/`.
2. Runs `cloudflare/wrangler-action@v3`.
3. Executes:

```bash
pages deploy content-site/cloudflare-pages \
  --project-name=abpiv-personal-brand \
  --branch=<current branch> \
  --commit-hash=<current sha>
```

Production deploys use the GitHub `production` environment. That environment has a required-reviewer gate for `BrewDogDev`, so pushes to `main` can pause until approved.

## Base URL Gotcha

Do not hard-code root-relative asset paths like:

```tsx
<img src="/img/headshot.png" />
```

That breaks under `/info/`. Use Docusaurus base URL helpers instead:

```tsx
import useBaseUrl from '@docusaurus/useBaseUrl';

const avatarUrl = useBaseUrl('/img/headshot.png');
```

The homepage hero image was updated this way in `src/components/HomeHero/index.tsx`.
